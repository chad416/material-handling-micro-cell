# MHMC Documentation Template Index

Use these templates as controlled starting points for project documentation.
Each template lists mandatory sections and guidance prompts. Replace prompts
with project-specific content before issuing any document for review, FAT, SAT,
or as-built release.

| Template | Purpose |
| --- | --- |
| `machine_functional_specification_template.md` | Machine purpose, scope, functions, modes, performance, and acceptance criteria. |
| `sequence_of_operations_template.md` | Normal, abnormal, manual, maintenance, jam recovery, and shutdown sequences. |
| `interlock_matrix_template.md` | Required permissives, inhibits, trip actions, reset rules, and bypass controls. |
| `io_list_template.md` | Symbolic I/O list, field device data, signal type, fail state, and mapping status. |
| `alarm_philosophy_template.md` | Alarm priority, acknowledgement, latching, reset, shelving, and event logging rules. |
| `hmi_screen_index_template.md` | Screen inventory, user roles, critical tags, commands, and navigation. |
| `network_topology_template.md` | Industrial network architecture, addressing, security, protocols, and diagnostics. |
| `fat_protocol_template.md` | Factory acceptance test prerequisites, test cases, evidence, and sign-off. |
| `sat_protocol_template.md` | Site acceptance test prerequisites, commissioning tests, evidence, and handover. |
| `punch_list_log_template.md` | Open issue tracking, ownership, priority, target dates, and closeout evidence. |
| `as_built_revision_log_template.md` | Controlled as-built changes, approvals, affected files, and release history. |

## Document Control Guidance

- Assign every issued document a document number, revision, owner, approver,
  status, and release date.
- Record source files and generated outputs used to create the document.
- Keep draft notes separate from approved requirements.
- Do not close FAT/SAT items without objective evidence: screenshot, log,
  measurement, test record, or signed field observation.
